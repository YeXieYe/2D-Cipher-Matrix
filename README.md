# 🛡️ 2D-Cipher-Matrix

**2D-Cipher-Matrix** — это учебный проект на Python для потокового симметричного шифрования числовых данных с последующим выводом криптограммы в виде двумерной ASCII-матрицы. Проект включает в себя дешифратор, способный автоматически определять угол поворота матрицы и очищать ее от текстового мусора, а также скрипт с тестами.

## <a id="цель-проекта"></a>🎯 Цель проекта
Разработать криптографический алгоритм, устойчивый к взлому с помощью машинного обучения и нейросетей.

Главная задача состояла в том, чтобы зашифровать данные и визуализировать их таким образом, чтобы нейросеть не смогла распознать алгоритм и расшифровать число — даже если скормить ей тренировочный датасет из большого количества тестовых матриц с правильными ответами.


## <a id="описание-проекта"></a>💡 Описание проекта

Проект объединяет потоковое шифрование на базе генератора псевдослучайных чисел и концепции визуального двухмерного кодирования. Данные не просто зашифровываются, но и упаковываются в визуальный контейнер, защищенный от смещений, текстового мусора и поворотов.

**Ключевые особенности:**

* **Симметричное шифрование:** Скрытие полезной нагрузки и ее длины с помощью динамического ключа.
* **Защита от нейросетей:** Алгоритм генерирует визуальный белый шум, лишенный статистических паттернов, что делает бесполезными любые попытки взлома с помощью машинного обучения (даже при наличии огромной тренировочной базы).
* **Автоматическая фильтрация при чтении файла:** Игнорирует любой посторонний текст, рамки терминала и съехавшие пробелы.
* **Автоповорот:** Способность дешифратора читать матрицу, повернутую на 90°, 180° или 270°.

## <a id="алгоритм-шифратора-encryptor"></a>⚙️ Алгоритм Шифратора (Encryptor)

Процесс создания шифра разделен на криптографическую подготовку и визуальный рендеринг:

1. **Подготовка данных:** Исходное число переводится в бинарный формат. Генерируется случайный вектор инициализации (Nonce).
2. **Генерация Keystream:** На основе «Секретного ключа + Nonce» генератор выдает поток псевдослучайных битов.
3. **Шифрование (XOR):** Длина сообщения и сами данные побитово складываются (XOR) с потоковым ключом.
4. **Рендеринг (Послойный):**
* *Слой 1:* Отрисовка внешней черной рамки и белой защитной зоны.
* *Слой 2:* Размещение углового маркера ориентации в левом верхнем углу.
* *Слой 3:* Отрисовка защищенного центрального блока, в который укладываются открытый Nonce и зашифрованная длина.
* *Слой 4:* Заполнение свободного пространства («Ядра») зашифрованными битами данных. Оставшиеся пустые пиксели заливаются криптографическим шумом для скрытия реального объема информации.



## <a id="алгоритм-дешифратора-decryptor"></a>🔍 Алгоритм Дешифратора (Decryptor)

Дешифратор работает как оптический сканер, восстанавливающий данные из шифра с криптографическим шумом:

1. **Очистка:** Чтение `cipher.txt`. Строки с буквами или текстовым мусором удаляются. Из оставшихся строк вычисляются строгие координаты рамки штрихкода.
2. **Снятие защиты:** Удаление двух внешних слоев (черной рамки и пустого отступа).
3. **Поиск ориентации:** Сканирование верхнего левого угла. Если маркер ориентации не найден, матрица топологически поворачивается на 90° по часовой стрелке (до 4 раз).
4. **Извлечение метаданных:** Чтение Nonce и зашифрованной длины из центрального блока.
5. **Дешифровка (XOR):** Генерация идентичного потокового ключа (Keystream). Дешифровка длины, сбор нужного количества пикселей с холста и их финальная расшифровка.

## <a id="почему-это-работает"></a>🧠 Почему это работает (почему нейросети бессильны)

Если вы попытаетесь обучить нейросеть, скормив ей датасет из множества пар *"сгенерированная картинка шифра ↔ правильное число"*, модель полностью провалится и выдаст точность случайного угадывания. 

Архитектура проекта математически блокирует любые попытки ML-анализа и взлома по четырем причинам:

1. **Отсутствие градиента:** Нейросети обучаются методом градиентного спуска. Алгоритму нужно видеть, что с каждой эпохой он хоть на долю процента приближается к правильному ответу. Но в алгоритмах с использованием криптографического сида и операции XOR нет состояния "почти угадал". Если нейросеть ошибется в ключе хотя бы на один бит, результат будет не "немного размытым", а нечитаемым белым шумом. ИИ просто не за что зацепиться, чтобы корректировать веса.
2. **Динамическая энтропия:** Алгоритм использует динамический открытый вектор инициализации (Nonce). Если вы зашифруете одно и то же число с одним и тем же паролем множество раз, алгоритм выдаст множество абсолютно разных визуальных матриц. Нейросеть физически не сможет выучить визуальный паттерн числа, потому что каждый раз оно выглядит как совершенно новый набор случайных пикселей.
3. **Абсолютный камуфляж данных:** Алгоритм не просто закрашивает пустые клетки случайным шумом, он шифрует саму длину сообщения. ИИ, смотрящий на матрицу, видит только хаос из черных и белых блоков. Он не знает, спрятано там маленькое число или гигантское, так как камуфляжный мусор и полезные данные визуально и статистически неразличимы.
4. **Сингулярность XOR:** Операция `(Текст XOR Белый шум) = Белый шум`. Наложение псевдослучайного ключа на исходные данные уничтожает абсолютно все статистические аномалии языка или бинарной структуры. В шифре не остается никаких закономерностей, которые алгоритмы глубокого обучения могли бы выявить и использовать для предсказаний.


## <a id="требования"></a>🛠 Требования
Проект написан на Python и не требует установки сторонних зависимостей.
* **Python:** 3.6 или выше.
* **Используемые модули:** `math`, `random` (стандартные библиотеки).


## <a id="использование"></a>🚀 Использование

**1. Шифрование:**
Запустите `encryptor.py`. Введите секретный ключ и число. Скопируйте полученную в консоли матрицу.

```bash
python encryptor.py

```

**2. Передача:**
Вставьте скопированную матрицу в файл `cipher.txt`. Файл может содержать любой посторонний текст сверху или снизу (логи, комментарии, слова) — дешифратор автоматически их проигнорирует.

**3. Дешифровка:**
Запустите `decryptor.py`. Укажите путь к файлу (`cipher.txt`) и введите тот же секретный ключ.

```bash
python decryptor.py

```

**4. Тесты:**
Запустите `tests.py`. Скрипт автоматически сгенерирует 8 заранее заданных тестов, 100 со случайными значениями, и еще 3 с разным углом поворота шифра.

```bash
python tests.py

```
