# 🛡️ 2D-Cipher-Matrix

**2D-Cipher-Matrix** — это учебный проект на Python для потокового симметричного шифрования числовых данных с последующим выводом криптограммы в виде двумерной ASCII-матрицы. Проект включает в себя дешифратор, способный автоматически определять угол поворота матрицы и очищать ее от текстового мусора, а также скрипт с тестами.

## <a id="цель-проекта"></a>🎯 Цель проекта
Разработать криптографический алгоритм, устойчивый к взлому с помощью машинного обучения и нейросетей.

Главная задача состояла в том, чтобы зашифровать данные и визуализировать их таким образом, чтобы нейросеть не смогла распознать алгоритм и расшифровать число — даже если скормить ей тренировочный датасет из большого количества тестовых матриц с правильными ответами.


## <a id="описание-проекта"></a>💡 Описание проекта

Проект объединяет потоковое шифрование на базе динамической криптографической маски (через операции XOR) и концепции визуального двухмерного кодирования. Данные не просто зашифровываются, но и упаковываются в визуальный контейнер, защищенный от смещений, текстового мусора и поворотов.

**Ключевые особенности:**

* **Симметричное шифрование:** Скрытие полезной нагрузки и ее длины с помощью динамического ключа.
* **Защита от нейросетей:** Алгоритм генерирует визуальный белый шум, лишенный статистических паттернов, что делает бесполезными любые попытки взлома с помощью машинного обучения (даже при наличии огромной тренировочной базы).
* **Автоматическая фильтрация при чтении файла:** Игнорирует любой посторонний текст, рамки терминала и съехавшие пробелы.
* **Автоповорот:** Способность дешифратора читать матрицу, повернутую на 90°, 180° или 270°.

## <a id="алгоритм-шифратора"></a>⚙️ Алгоритм Шифратора (Encryptor)

Процесс создания шифра разделен на криптографическую подготовку и визуальный рендеринг:

1. **Подготовка данных:** Исходное число переводится в бинарный формат. Генерируется случайный вектор инициализации (Nonce).
2. **Создание маски:** Секретный ключ и Nonce циклично растягиваются до нужной длины и побитово складываются (XOR) друг с другом, образуя уникальную криптографическую маску.
3. **Шифрование (XOR):** Длина сообщения и сами данные побитово складываются (XOR) с полученной маской.
4. **Рендеринг (Послойный):**
* *Слой 1:* Отрисовка внешней черной рамки и белой защитной зоны.
* *Слой 2:* Размещение углового маркера ориентации в левом верхнем углу.
* *Слой 3:* Отрисовка защищенного центрального блока, в который укладывается только зашифрованная длина исходного числа.
* *Слой 4:* Заполнение свободного пространства («Ядра») зашифрованными битами данных вперемешку с инвертированными битами Nonce (каждый 4-й бит). Оставшиеся пустые пиксели заливаются криптографическим шумом для скрытия реального объема информации.



## <a id="алгоритм-дешифратора"></a>🔍 Алгоритм Дешифратора (Decryptor)

Дешифратор работает как оптический сканер, восстанавливающий данные из шифра с криптографическим шумом:

1. **Очистка:** Чтение `cipher.txt`. Строки с буквами или текстовым мусором удаляются. Из оставшихся строк вычисляются строгие координаты рамки штрихкода.
2. **Снятие защиты:** Удаление двух внешних слоев (черной рамки и пустого отступа).
3. **Поиск ориентации:** Сканирование верхнего левого угла. Если маркер ориентации не найден, матрица топологически поворачивается на 90° по часовой стрелке (до 4 раз).
4. **Извлечение метаданных:** Чтение зашифрованной длины из центрального блока и извлечение битов Nonce из основного визуального потока (каждый 4-й бит).
5. **Дешифровка (XOR):** Восстановление криптографической маски (путем повторного растягивания и сложения ключа с извлеченным Nonce). Дешифровка длины, отсечение визуального шума и финальная расшифровка самого числа.

## <a id="почему-это-работает"></a>🧠 Почему это работает (почему нейросети бессильны)

Если вы попытаетесь обучить нейросеть, скормив ей датасет из множества пар "сгенерированная картинка шифра ↔ правильное число", модель полностью провалится и выдаст точность случайного угадывания.

Архитектура проекта математически блокирует любые попытки ML-анализа и взлома по четырем причинам:

1. **Отсутствие градиента:** Нейросети обучаются методом градиентного спуска. Алгоритму нужно видеть, что с каждой эпохой он хоть на долю процента приближается к правильному ответу. Но в алгоритмах, построенных на строгих побитовых масках и операции XOR, нет состояния «почти угадал». Если нейросеть ошибется в восстановлении ключа или смещения хотя бы на один бит, итоговая маска «разрушится», и результат будет не «немного размытым числом», а нечитаемым хаосом. ИИ просто не за что зацепиться, чтобы корректировать веса.
2. **Динамическая энтропия маски:** Алгоритм использует случайный вектор инициализации (Nonce), который циклично вплетается в саму структуру шифрующей маски. Если вы зашифруете одно и то же число с одним и тем же паролем сотню раз, алгоритм сгенерирует сотню абсолютно разных криптографических масок и визуальных матриц. Нейросеть физически не сможет выучить «внешний вид» числа.
3. **Рассеивание: Алгоритм не просто** закрашивает пустые клетки случайным шумом и шифрует длину сообщения. Он структурно перемешивает служебные биты (Nonce) с битами зашифрованного числа прямо в визуальном потоке. ИИ, смотрящий на матрицу, видит неразрывный хаос: он не знает, какова длина числа, где заканчиваются полезные данные, где спрятаны биты вектора инициализации, а где начинается просто сгенерированный мусор.
4. **Сингулярность XOR:** Операция (Данные XOR Уникальная маска) = Белый шум. Побитовое наложение комбинированной маски (Секретный ключ + Nonce) на исходные данные маскирует все статистические и бинарные закономерности. В финальном шифре не остается никаких визуальных паттернов, которые алгоритмы машинного обучения могли бы выявить и использовать для предсказаний.


## <a id="требования"></a>🛠 Требования
Проект написан на Python и не требует установки сторонних зависимостей.
* **Python:** 3.6 или выше.
* **Используемые модули:** `math`, `random` (стандартные библиотеки).


## <a id="использование"></a>🚀 Использование

**1. Шифрование:**
Запустите `encryptor.py`. Введите секретный ключ и число. Скопируйте полученную в консоли матрицу.

```bash
python ../src/encryptor.py

```

**2. Передача:**
Вставьте скопированную матрицу в файл `cipher.txt`. Файл может содержать любой посторонний текст сверху или снизу (логи, комментарии, слова) — дешифратор автоматически их проигнорирует.

**3. Дешифровка:**
Запустите `decryptor.py`. Укажите путь к файлу (`cipher.txt`) и введите тот же секретный ключ.

```bash
python ../src/decryptor.py

```

**4. Тесты:**
Запустите `tests_generator.py`. Скрипт автоматически сгенерирует 8 заранее заданных тестов, 100 со случайными значениями, и еще 3 с разным углом поворота шифра.

```bash
python ../src/tests_generator.py

```
